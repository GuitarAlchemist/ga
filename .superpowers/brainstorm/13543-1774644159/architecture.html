Section 1: Directory Layout & Junction Architecture

The physical structure on disk

Directory Structure — ga/
ga/
├── Apps/ga-server/GaApi/
│ ├── bin/
│ │ ├── blue/net10.0/ ← physical dir, 153+ DLLs
│ │ ├── green/net10.0/ ← physical dir, 153+ DLLs
│ │ ├── active → blue/ or green/ ← NTFS junction point
│ │ └── .slot-state ← JSON: which slot is active, last swap time
│ └── ...
├── Common/GA.*/ ← library projects (same pattern)
├── Scripts/
│ ├── ga-build.ps1 ← builds to inactive slot
│ ├── ga-swap.ps1 ← moves junction, restarts server
│ ├── ga-start.ps1 ← starts from active slot
│ ├── ga-rollback.ps1 ← emergency swap to other slot
│ └── ga-status.ps1 ← shows slot health, SAC trust
├── Directory.Build.props ← OutputPath override
└── .slot-state.json ← solution-level slot tracking

How It Works

🔵 Blue Slot (active)

  • Server runs from here
  • SAC trusts these binaries
  • Never touched during build
  • Junction active → blue/

🟢 Green Slot (inactive)

  • Build output goes here
  • DLLs aging, gaining SAC trust
  • Health-checked before swap
  • Becomes active after swap

Swap Sequence

1. Stop server
→
2. Remove junction
→
3. Re-create junction → green
→
4. Health check
→
5. Start server
→
6. If fail → rollback to blue

Key Detail: MSBuild OutputPath

Directory.Build.props reads .slot-state.json to determine the inactive slot, then sets OutputPath accordingly. This means dotnet build anywhere in the solution automatically targets the right slot — no special flags needed.
