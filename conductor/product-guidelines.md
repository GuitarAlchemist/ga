# Product Guidelines & Conventions

## Code Style
- **C#:** Follow `.editorconfig`. 
  - 4-space indentation.
  - File-scoped namespaces.
  - PascalCase for types/methods.
  - camelCase for locals/parameters.
- **TypeScript/React:**
  - Functional components.
  - PascalCase for filenames/components.
  - Use `npm run lint`.

## Documentation
- Maintain high-quality documentation in `docs/` and `Specs/`.
- Audit for sensitive information before committing.
