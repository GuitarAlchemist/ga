# Deletion-deck set-reconstruction in Z_N

## Definitions

Work in Z_N (pitch classes mod N), classes taken under the dihedral action
(transposition + inversion, "Tn/TnI"); write SC(S) for the class of S.
For |S| ≥ 3 define the **deletion deck** of S as the *set* (no
multiplicities) of classes of its one-note deletions:

    D(S) = { SC(S \ {x}) : x ∈ S }.

D is a class invariant (D depends only on SC(S)). The question is whether it
is a **complete** invariant.

## Statement

**Theorem (trichord case, proved below).** For N ≥ 6 and |S| = 3:
D determines SC(S) **iff 5 ∤ N**. When N = 5d, the unique collision is

    A = {0, d, 2d}  vs  B = {0, d, 3d}   (gap multisets {d,d,3d} vs {d,2d,2d}),

which share D(A) = D(B) = { SC({0,d}), SC({0,2d}) } while SC(A) ≠ SC(B).

**Conjecture (general case).** For N ≥ 6 and |S| ≥ 4, D determines SC(S) for
*every* N — i.e. the mod-5 trichord pair is the **only** failure of
deletion-deck set-reconstruction. Verified exhaustively for all 6 ≤ N ≤ 20
(full sweep of all subsets, all cardinalities): the failing moduli are
exactly N = 10, 15, 20, each with the single predicted trichord collision and
**no collision at any cardinality ≥ 4**.

## Proof — trichord case

A trichord corresponds, up to the dihedral action, to its multiset of cyclic
gaps {x, y, z}, x + y + z = N, x, y, z ≥ 1 (any permutation of three gaps is
realizable by rotation/reflection). Deleting the vertex incident to gaps x
and y merges them into a dyad of span x + y ≡ −z, of class ic(z), where
ic(g) = min(g, N − g). Hence

    D(trichord with gaps {x,y,z}) = { ic(x), ic(y), ic(z) }   (as a set).

A collision is a pair of *distinct* gap multisets with equal ic-sets S.

*Step 1 — each ic-multiset admits at most one gap multiset.* Given an
ic-multiset {v1, v2, v3}, each gap is gi = vi or N − vi; write k for the
number of "flipped" gaps (gi = N − vi). Summing to N forces:
k = 0 with v1+v2+v3 = N; or k = 1 (flip at j) with vj = Σ_{i≠j} vi; k ≥ 2 is
impossible (it forces some vi ≤ 0 or Σvi ≥ 2N against vi ≤ N/2). If both a
k = 0 and a k = 1 solution exist, then Σvi = N and 2vj = Σvi give
vj = N/2 = N − vj, so the "two" solutions are the same multiset; two k = 1
solutions at j ≠ j′ force vj = vj′ and again coincide. Hence: one
ic-multiset → at most one gap multiset.

*Step 2 — collisions need |S| = 2 and split multiplicities.* By Step 1,
distinct gap multisets sharing the ic-*set* S must have **different
ic-multisets** over the same support S. For |S| = 3 or |S| = 1 the multiset
is forced by the set, so no collision. For S = {p, q} the two candidate
ic-multisets are {p,p,q} and {p,q,q}.

*Step 3 — realizability forces N = 5d.* By the k-analysis, {p,p,q} is
realizable iff (2p + q = N) or (q = 2p, gaps {p, p, N−2p}); symmetrically
{p,q,q} iff (p + 2q = N) or (p = 2q, gaps {q, q, N−2q}). A collision needs
both realizable with p ≠ q:
- 2p+q = N and p+2q = N ⟹ p = q, excluded;
- 2p+q = N and p = 2q ⟹ **N = 5q**: gaps {2q,2q,q} vs {q,q,3q};
- q = 2p and p+2q = N ⟹ **N = 5p**: the same pair, mirrored;
- q = 2p and p = 2q ⟹ p = q = 0, excluded.
So collisions exist iff 5 | N, and the unique colliding pair is
{d,d,3d} vs {d,2d,2d} (d = N/5), i.e. {0,d,2d} vs {0,d,3d}. These are
inequivalent since their interval multisets {d,d,2d} ≠ {d,2d,2d} differ. ∎

## Theorem — tetrachord multiplicity recovery (O3, proved 2026-07-13)

**Theorem T4.** For every N ≥ 4 and every S ⊂ Z_N with |S| = 4, the set-deck
D(S) (together with N) determines the multiplicity vector of the multiset-deck
M(S). In fact the Kelly-parity system has a **unique** admissible solution for
every realizable support, with the complete coincidence classification below.
(At n = 4 the deletions of a card — a trichord — are exactly its ICV, so Kelly
parity and ICV-integrality are the *same* system: T4 is precisely the n = 4
instance of open link O1.)

Setup. Write S as its cyclic gap word (g₁, g₂, g₃, g₄), gᵢ ≥ 1, Σgᵢ = N
(necklace dictionary, Lemma L5 of the referee report). The four cards are the
adjacent fusions, as gap multisets:

    F₁ = {g₁+g₂, g₃, g₄}   F₂ = {g₂+g₃, g₄, g₁}
    F₃ = {g₃+g₄, g₁, g₂}   F₄ = {g₄+g₁, g₂, g₃}

and a trichord class is exactly its gap multiset (any 3-letter cyclic word is
rotation/reflection-equivalent to any permutation of it).

**Lemma T4.1 (adjacent coincidence).** F₁ = F₂ ⟺ g₁ = g₃.
*Proof.* (⇐) immediate substitution. (⇒) The element g₁+g₂ of F₁ must occur in
F₂ = {g₂+g₃, g₄, g₁}. It cannot equal g₁ (g₂ ≥ 1). If g₁+g₂ = g₂+g₃ then
g₁ = g₃. If g₁+g₂ = g₄: removing one copy of the value g₄ from both multisets
leaves {g₃, g₄} = {g₂+g₃, g₁}; g₂+g₃ ≠ g₃ forces g₂+g₃ = g₄ and g₁ = g₃ — so
g₁ = g₃ in every case. ∎
By the same computation F₃ = F₄ ⟺ g₁ = g₃: **adjacent coincidences come in
simultaneous pairs** (and cyclically, F₂ = F₃ ⟺ F₄ = F₁ ⟺ g₂ = g₄).

**Lemma T4.2 (opposite coincidence).** F₁ = F₃ ⟺ g₁+g₂ = g₃+g₄ = N/2 and
{g₁, g₂} = {g₃, g₄}.
*Proof.* (⇐) immediate. (⇒) The fused part g₁+g₂ strictly exceeds g₁ and g₂,
so its match in F₃ = {g₃+g₄, g₁, g₂} must be g₃+g₄; with g₁+g₂ + g₃+g₄ = N
this gives both equal N/2, and removing that common value leaves
{g₃, g₄} = {g₁, g₂}. ∎

**Proposition T4.3 ((3,1) is impossible).** If three of the four cards
coincide, all four do.
*Proof.* Any 3-subset of {F₁..F₄} contains two adjacent-coincident pairs
covering both conditions of T4.1 (e.g. {F₁,F₂,F₃} gives F₁=F₂ and F₂=F₃), or
one adjacent pair plus an opposite pair which, by T4.2's multiset condition
combined with the adjacent equality, again forces both g₁ = g₃ and g₂ = g₄.
Then the necklace is (a, b, a, b) and all four fusions equal {a+b, a, b}. ∎

**Proposition T4.4 (classification).** Up to rotation/reflection of the
necklace:

| r | necklace | m | notes |
|---|---|---|---|
| 4 | generic | (1,1,1,1) | trivially unique |
| 3 | (a, b, b, a), a ≠ b | (2,1,1) | N = 2(a+b); doubled card = {N/2, a, b} |
| 2 | (a, b, a, d), b ≠ d | (2,2) | (3,1) excluded by T4.3 |
| 1 | (a, b, a, b) | (4) | trivially unique |

*Proof.* r = 3 means exactly one coincident pair. By T4.1 it cannot be
adjacent (those come in pairs), so it is opposite: T4.2 with g₁ ≠ g₃ (else an
adjacent coincidence too) forces g₁ = g₄, g₂ = g₃ — the necklace (a, b, b, a),
a ≠ b. Direct computation: F₁ = F₃ = {N/2, a, b}, F₂ = {2b, a, a},
F₄ = {2a, b, b}, and F₂ = F₄ would force a = b. For r = 2, the partition into
two pairs is adjacent/adjacent (T4.1, necklace (a,b,a,d), b ≠ d; the two
classes {a+b, a, d} ≠ {a+d, a, b} since matching a+b to a+d, a or b each
forces b = d, b = 0 or a = 0); opposite/opposite is impossible (T4.2 for both
pairs forces {g₂,g₃} = {g₄,g₁} with g₂ = g₁, g₃ = g₄ and then a = b). ∎

**Uniqueness of the parity solution.** Non-uniqueness requires an alternative
admissible m′; δ = m − m′ has Σδ = 0 and Σ_{δᵢ odd} ICV(Cᵢ) ≡ 0 (mod 2), i.e.
equal mod-2 ic-signatures on the odd-δ support.

- r = 1, 4: only one admissible m. ∎
- r = 3, m = (2,1,1): every alternative is another permutation of (2,1,1), so
  δ = e_{doubled} − e_{single} — it needs sig(doubled) = sig(single). From
  T4.4: the doubled card {N/2, a, b} has ic-multiset {N/2, a, b} (a, b < N/2,
  all distinct) — signature of **size 3**; each single {2b, a, a} / {2a, b, b}
  has ic-multiset with at most two distinct values — signature of **size 1**
  ({ic(2b)} or, if ic(2b) = a, {a}). Sizes differ; no alternative. ∎
- r = 2: (3,1)/(1,3) are never the true m (T4.3), so m = (2,2) and the
  alternatives (3,1), (1,3) need sig(C₁) = sig(C₂), i.e. the multiset
  {ic(a+b), ic(a+d), ic b, ic d} ⊎ {ic a, ic a} must be even in every value.
  Since ic(a+b) = ic(a) would force a+b ≡ ±a, i.e. b = 0 or d = 0 (as
  a+b = N−a ⟺ d = 0 from N = 2a+b+d), the two copies of ic a stand alone and
  {ic(a+b), ic(a+d), ic b, ic d} must pair up internally. All three matchings
  die on positivity: ic b = ic d ⟹ b = d or b+d = N (impossible inside one
  necklace); ic(a+b) = ic b ∧ ic(a+d) = ic d ⟹ a+2b = N = a+2d ⟹ b = d;
  ic(a+b) = ic d ∧ ic(a+d) = ic b ⟹ (a+b = d ∧ a+d = b, adding: a = 0 — the
  flip branches a+b = N−d, a+d = N−b give a = 0 directly). The mixed
  patterns (two of the four equal to ic a) die because ic(a+b) = ic(a) and
  ic(a+d) = ic(a) are impossible as shown, and ic b = ic d = ic a puts
  {b, d} = {a, N−a} with N−a not a valid co-gap. So sig(C₁) ≠ sig(C₂). ∎

## Position against adjacent work (per the 2026-07-13 literature pass)

- **Lewin's inclusion functions** (EMB/COV; JMT 21/2 1977, PNM 18 1979–80)
  and the n-class vectors (Buchler) retain **multiplicities** of subset
  classes; this result uses only the *support* of the deletion layer — a
  strictly weaker datum — and is not implied by those tools.
- **Harary's Set Reconstruction Conjecture** (1964) concerns unweighted
  simple graphs under isomorphism; a pc-set is a complete cyclic-metric
  graph under the dihedral action, and no deck-preserving reduction between
  the categories is known. **No claim is made about general graph
  reconstruction** — this is a reconstruction theorem for the specific
  family of cyclic metric pitch-class structures, notable precisely because
  it is *false* in general (mod-5 family) yet true for Z12.
- The Z12 case (music-theoretic universe) is the culturally relevant
  instance: **every Z12 set class of cardinality ≥ 3 is determined by the
  bare set of its one-note-deletion classes.**
