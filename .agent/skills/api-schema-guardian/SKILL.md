# API Schema Guardian

## Role
Catch mismatches between what a controller _declares_ it returns and what it _actually_ returns. Also enforce request-model validation attributes (`[Required]`, `[Range]`, `[StringLength]`) so ASP.NET Core model binding rejects bad input before it reaches business logic.

---

## 1. Declaration vs Reality Check

### Rule 1 — Declared type must match ActionResult type

```csharp
// VIOLATION: declared typeof(Foo) but returns Bar
[ProducesResponseType(typeof(Foo), 200)]
public ActionResult<Bar> Get() => Ok(new Bar());  // ← mismatch

// CORRECT
[ProducesResponseType(typeof(Bar), 200)]
public ActionResult<Bar> Get() => Ok(new Bar());
```

**How to find violations:**
```bash
grep -n "ProducesResponseType(typeof" Apps/ga-server/GaApi/Controllers -r --include="*.cs"
# Then cross-reference each with the method signature on the adjacent lines.
```

### Rule 2 — ActionResult<T> generic parameter must match ProducesResponseType typeof

When a method is `ActionResult<IEnumerable<ChordInContext>>`, the attribute must be `[ProducesResponseType(typeof(IEnumerable<ChordInContext>), 200)]` — not `typeof(ChordInContext)`.

### Rule 3 — IActionResult methods need explicit ProducesResponseType

`IActionResult` erases the return type from OpenAPI. Every `IActionResult`-returning method must declare all possible response types explicitly.

---

## 2. Request Model Validation

For every model used as a `[FromBody]` or `[FromQuery]` parameter:

| Property characteristic | Required attribute |
|-------------------------|-------------------|
| Must not be null/empty | `[Required]` |
| String with max length | `[StringLength(max)]` |
| Numeric in a range | `[Range(min, max)]` |
| Enum parameter | `[EnumDataType(typeof(MyEnum))]` |

**Quick scan:**
```bash
# Find [FromBody] parameters — check their model type for [Required] attributes
grep -rn "\[FromBody\]" Apps/ga-server/GaApi/Controllers --include="*.cs"
```

---

## 3. Swagger/OpenAPI Consistency

The GaApi exposes its spec at `/swagger/v1/swagger.json` (dev only).

Verify:
1. Every controller is tagged in `SwaggerExtensions.cs` with the correct tag group.
2. Every operation ID is unique (Swagger warns on duplicates).
3. All enum types are serialised as strings in the spec (not integers).

Check `Apps/ga-server/GaApi/Extensions/SwaggerExtensions.cs` for tag assignments.

---

## 4. Nullable Alignment

With `<Nullable>enable</Nullable>`:
- A `string?` property in a response model appears as `nullable: true` in OpenAPI.
- A non-nullable `string` must be `nullable: false` (the default).
- Ensure response records use `?` only where the field is genuinely optional.

---

## 5. Workflow

1. Pick an `open` schema-guardian item from BACKLOG.
2. Read the controller + the referenced model files.
3. Cross-check declarations against reality using the rules above.
4. If there's a mismatch: fix the `[ProducesResponseType]` attribute (or the model) — do NOT change business logic.
5. Run `pwsh Scripts/api-quality-check.ps1`.
6. Update BACKLOG: `done`.
